normalization.interpolate(data1, data2)¶

Interpolates data2 into the locations of data1. Requires data1 and data2 have no less than two values. If data1 and data2 have more than two columns, this will use the first and last columns.

Parameters

  • data1 – Data you want to manipulate in the future

  • data2 – Data you want to interpolate to be happy with data1.

Returns interpolated_data2

data2 which has been interpolated to match the x-indices of data1.

normalization.normalize_pandas(data1, data2, operation=<ufunc 'multiply'>, operation_args=(), operation_kwargs={}, new_units='')¶

Performs an operation on data1 and data2, interpolating data from data2 as needed, and applying the operation specified (can be any function that takes two numpy arrays). New data will have indices and units of data1

Parameters

  • data1 – Main data to be manipulated/plotted/used

  • data2 – Data to normalize / multiply / operate on relative to data1

  • operation – operation function i.e. np.multiply. Operates such that new_data = data1 [OPERATION] data2, i.e. new_data = data1 / data2

  • operation_args – Optional positional arguments to be passed into operation

  • operation_kwargs – Optional keyword arguments to be passed into operation

  • new_units – New column name for unit-laden dataFrame

Returns new_data

pd.DataFrame = data1 [OPERATION] data2

plotters.default_plot_pandas(data, fig=None, ax=None, theory_func=None, theory_kw={}, subplot_kw={}, line_kw={})¶

Plots a pandas DataFrame, assuming the xdata is located in the first column and the ydata is located in the second column.

Parameters

  • data – DataFrame to be plotted.

  • fig – Figure to plot the data to

  • ax – axes to plot the data to

  • theory_func – Function to plot along with xdata, of the form theory_func(xdata, theory_kw)

  • theory_kw – Keyword arguments to be passed into theory_func

  • subplot_kw – Keyword arguments to be passed into fig.subplots()

plotters.default_plotter(data, fig=None, ax=None, ydata=None, theory_func=None, theory_kw={}, line_kw={}, subplot_kw={})¶

Default plotter which handles plotting pandas DataFrames, numpy arrays, and regular ol data.

  • theory_kw – Keyword arguments to pass into theory_func

  • line_kw – Keyword arguments to pass into ax.plot() function

  • subplot_kw – Keyword arguments to pass into fig.subplots() function

Implementation of powerSpectrumPlot for a pandas DataFrame. Plots a given power spectrum with units in the form Unit Name (unit type), i.e. Photocurrent (mA).

Parameters

  • power_spectrum – The power spectrum to be plotted, with frequency bins on one column and power in the second column

  • line_kw – Keyword arguments to parameterize line in call to ax.plot()

  • fig – (optional) Figure to plot the data to

  • ax – (optional) axes to plot the data to

  • line_kw – Keyword arguments to pass into ax.plot()

  • subplot_kw – Keyword arguments to pass into fig.subplots()

  • theory_func – Theoretical PSD function

  • theory_kw – Keyword arguments to pass into theory_func

plotters.reflectance_plotter(photocurrent, reference_photocurrent, R_ref, fig=None, ax=None, theory_func=None, theory_kw={}, subplot_kw={}, line_kw={})¶

Plotter which takes a photocurrent, normalizes it to a reference photocurrent, and multiplies that be the reference’s known or theoretical reflectance.

Parameters

  • photocurrent – Pandas DataFrame of measured photocurrent vs. wavelength (or frequency)

  • reference_photocurrent – Pandas DataFrame of measured photocurrent reflecting from a reference surface with a known reflectance

  • R_ref – Pandas DataFrame of known reflectance of surface (theoretical or measured)

  • fig – Optional figure to plot to. If empty, creates a figure.

  • ax – Optional axes to plot to. If empty, creates a new axes

  • theory_func – Theoretical reflectance function to plot alongside the measured reflectance

  • theory_kw – Keyword arguments for theoretical plotting function

  • subplot_kw – Keyword argumets to pass into the .subplots() function during Axes creation.

  • line_kw – Keyword arguments to pass into the .plot() function during Line2D creation.
